Configuration

Set up your game IDs and access restrictions

🎮 Main Game Place ID

The Roblox Place ID of the game players should be launched into. Found in the URL: roblox.com/games/YOUR_ID/...

🔀 Redirect Game Place ID

Players without a valid ID will be teleported to this game instead of kicked. Recommended: a waiting room or public game.

🔐 Access Code Requirement

If enabled, players must enter a valid access code during registration. Add multiple codes separated by commas.

🔑 API Secret Key

Use this key in your Roblox Lua script when making HTTP requests to the verification API. Keep it secret!

⚠️ Danger Zone

Permanently delete all registration data. This cannot be undone.

API Guide

How to connect your Roblox game to this website

📌 Important Note

This launcher uses localStorage as its database — it works great for testing and small communities. For a large production game with many concurrent players, you would host this website on a real server with a database (e.g. Supabase, Firebase, or a Node.js backend). The Roblox Lua code below works the same way regardless.

Step 1 — Configure Place IDs

Go to the Configuration tab and fill in your Main Game Place ID and your Redirect Place ID (for players without a valid ID).

Step 2 — The Roblox Lua Script

Add this script to ServerScriptService in Roblox Studio. Replace the placeholder values with your actual website URL and API key.

Step 3 — Enable HTTP Requests

In Roblox Studio, go to Game Settings → Security and enable "Allow HTTP Requests". Without this, the Lua script cannot call your website's API.

Share this website with your players. Once registered, they click the Launch Roblox Game button — it automatically embeds their ID and opens Roblox.
